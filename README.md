# repo-wiki-viewer

[![CI](https://github.com/ht0x/repo-wiki-viewer/actions/workflows/ci.yml/badge.svg)](https://github.com/ht0x/repo-wiki-viewer/actions/workflows/ci.yml)

A documentation viewer for repository wiki documentation generated by the skill **`repo-wiki-standard`** from the [`ht0x/software-development-skills`](https://github.com/ht0x/software-development-skills) marketplace plugin.
It provides a much better reading experience for project documentation and Mermaid diagrams without requiring you to publish or export the documentation elsewhere.

<img width="1280" height="720" alt="1 - Demo" src=""/>

You can access it here: **https://ht0x.github.io/repo-wiki-viewer**

## Introduction

The idea for this project came from exploring **Qoder's Repo Wiki** feature on [`Qoder's Repo Wiki`](https://docs.qoder.com/user-guide/repo-wiki). It presents an entire repository as a browsable wiki with rich visual diagrams directly inside the IDE.
However, Qoder's Repo Wiki is only available to paid users, and reading the generated files directly in an IDE or text editor has several drawbacks:

- In most text editors, Markdown is displayed as raw text.
- Some IDEs require additional Markdown extensions to render Mermaid diagrams, and even then the diagrams are often displayed at a small size that requires zooming in.

To provide a better experience:

- Generate repository documentation in wiki format using the [`repo-wiki-standard`](https://github.com/ht0x/software-development-skills) skill.
- View the generated documentation using **repo-wiki-viewer**.

## Getting Started

### 1. Install the plugin

Install the plugin [`ht0x/software-development-skills`](https://github.com/ht0x/software-development-skills) as a Claude Code plugin for either **Claude Code CLI**

Open Claude Code CLI and run these commands below:

```bash
/plugin marketplace add ht0x/software-development-skills
```

```bash
/plugin install software-development-ht0x@software-development-ht0x
```

```bash
/reload-plugins
```

**Video Instruction**

### 2. Generate the documentation

- Open your project in **Claude Code CLI**.
- Run the following command:

```text
/repo-wiki-standard
```

Then follow the prompts, including:

- Selecting the output directory (default: `docs/wiki/`)
- Choosing the documentation language

The skill will generate a complete repository wiki, including:

- `Home.md`
- Technical documentation such as architecture overviews, data flows, dependencies, and more
- Mermaid diagrams

**Video Instruction**
<video controls src="" title="Generate documentation"></video>

> **Cost Note**
>
> The `repo-wiki-standard` skill is designed to reduce token usage on each run by using an incremental generation mechanism that regenerates only the pages affected by source code changes. **This mechanism is only for commited Git repo**.
>
> However, even with this optimization, large projects may still consume **millions of input and output tokens** during documentation generation. Consider using more cost-effective LLM providers and models to reduce costs.

### 3. Open repo-wiki-viewer

You can access it here: **https://ht0x.github.io/repo-wiki-viewer/**

**Load the generated documentation**

- Click **Load Folder** and select the generated documentation folder in your project (default: `docs/wiki/`).
- Alternatively, simply **drag and drop the entire folder** onto the webpage.

The documentation, including all Mermaid diagrams, will be rendered directly in your browser.
All data is processed locally in the browser.

## Features

### 1. Open a documentation folder

- Click **Load Folder**, then select the documentation output folder generated by the `repo-wiki-standard` skill.
  The documentation will be rendered immediately.

### 2. Drag and drop support

- Drag the documentation folder from your file manager and drop it onto the webpage.

### 3. Reload

- Update the documentation in your repository by running the `repo-wiki-standard` skill again.
- Click **Reload** to reload the documentation directly from disk **without selecting the folder again**.
- The **Reload** feature is available only when the documentation was originally opened using **Load Folder**.

> **Browser Compatibility**
>
> The **Reload** feature uses the **File System Access API**.
>
> This API is currently supported only by Chromium-based browsers:
>
> - Chrome
> - Edge
>
> Firefox and Safari do not currently support this API, so the **Reload** feature is unavailable.
>
> In that case, simply reload the documentation by:
>
> - Clicking **Load Folder**,
> - Or dragging and dropping the folder again.
